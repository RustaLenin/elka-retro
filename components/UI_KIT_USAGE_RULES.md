# Правила использования UI-Kit компонентов

## Основной принцип

**Всегда используйте компоненты из UI-Kit для обеспечения стилистического единства проекта.**

UI-Kit создан для того, чтобы обеспечить единообразный внешний вид и поведение интерфейсных элементов по всему проекту. Использование нативных HTML элементов (`<button>`, `<input>`, `<select>` и т.д.) вместо компонентов UI-Kit нарушает это единство.

## Правила

### 1. Приоритет UI-Kit компонентов

Если нужный визуальный стиль или поведение недоступны в UI-Kit, это **повод доработать UI-Kit**, а не создавать собственные элементы.

**❌ Неправильно:**
```html
<button class="my-custom-button">Кнопка</button>
```

**✅ Правильно:**
```html
<ui-button type="primary" label="Кнопка"></ui-button>
```

### 2. Расширение UI-Kit

Если существующий компонент UI-Kit не поддерживает нужный вариант:

1. **Добавьте новый тип/модификатор** в существующий компонент UI-Kit
2. **Документируйте** новый вариант в README компонента
3. **Используйте** новый вариант во всех местах, где он нужен

**Пример:**
Если нужна кнопка с прозрачным фоном и border:
- Добавьте новый тип `type="ghost"` или модификатор в `ui-button`
- Обновите стили компонента
- Используйте `<ui-button type="ghost">` везде, где нужен такой стиль

### 2.1. Кастомизация через классы

**Важно:** Компоненты UI-Kit поддерживают кастомизацию через дополнительные классы, установленные через атрибут `class`.

**✅ Правильно:**
```html
<ui-button type="primary" class="my-custom-class">Кнопка</ui-button>
```

Компоненты UI-Kit **сохраняют дополнительные классы** при рендеринге, что позволяет:
- Добавлять специфичные стили для конкретных контекстов
- Использовать селекторы для точечной стилизации
- Не нарушать базовую функциональность компонента

**Правило для разработчиков UI-Kit:**
При реализации компонентов UI-Kit **всегда расширяйте классы**, а не перезаписывайте их полностью:

**❌ Неправильно:**
```javascript
render() {
  this.className = `ui-button ${type}`; // Перезаписывает все классы
}
```

**✅ Правильно:**
```javascript
render() {
  // Сохраняем дополнительные классы
  const preservedClasses = Array.from(this.classList).filter(cls => 
    cls !== 'ui-button' && cls !== 'primary' && cls !== 'secondary' // и т.д.
  );
  // Устанавливаем базовые классы
  this.className = `ui-button ${type}`;
  // Восстанавливаем сохраненные классы
  preservedClasses.forEach(cls => this.classList.add(cls));
}
```

### 3. Компоненты UI-Kit

Доступные компоненты UI-Kit:
- `ui-button` - кнопки всех типов
- `ui-icon` - иконки
- `ui-input-text` - текстовые поля ввода
- `ui-select-single` / `ui-select-multi` - выпадающие списки
- `ui-checkbox` - чекбоксы
- `ui-modal` - модальные окна
- `ui-notification` - уведомления
- `ui-loader` - индикаторы загрузки
- `ui-tooltip` - всплывающие подсказки
- `ui-form-controller` / `ui-form-field` - обёртки для форм

### 4. Исключения

Исключения из правила возможны только в следующих случаях:

1. **Семантические HTML элементы** без UI-компонента (например, `<nav>`, `<main>`, `<article>`)
2. **Временные решения** с явным TODO комментарием о необходимости замены на UI-Kit компонент
3. **Специфичные элементы**, которые не являются частью UI (например, контентные блоки)

### 5. Проверка перед коммитом

Перед коммитом проверьте:
- ✅ Все кнопки используют `ui-button`
- ✅ Все поля ввода используют компоненты из `ui-kit/form`
- ✅ Все иконки используют `ui-icon`
- ✅ Нет дублирования стилей кнопок/инпутов вне UI-Kit

## Примеры

### Кнопка удаления

**❌ Неправильно:**
```html
<button class="cart-item_remove-btn">
  <ui-icon name="trash"></ui-icon>
  <span>Убрать из корзины</span>
</button>
```

**✅ Правильно:**
```html
<ui-button type="ghost" icon="trash" label="Убрать из корзины"></ui-button>
```

### Поле ввода

**❌ Неправильно:**
```html
<input type="text" class="my-input" placeholder="Введите имя">
```

**✅ Правильно:**
```html
<ui-input-text name="name" placeholder="Введите имя"></ui-input-text>
```
