# Настройка пользовательских полей профиля (PODS)

Чтобы данные, которые запрашиваем на шагах оформления заказа и в настройках профиля, сохранялись в WordPress и были доступны в админке, нужно расширить сущность **User** через PODS.

## 1. Открываем Pod пользователя

1. Перейти в **PODS Admin → Edit Pods**.
2. Выбрать Pod с типом `User`.

## 2. Добавляем поля

| Поле | Тип | Название (Label) | Ключ (Name) | Описание |
| --- | --- | --- | --- | --- |
| Телефон | Text | `Телефон` | `phone_number` | Используется на шаге «Личные данные», также отображается в заказах |
| Имя | Text | `Имя` | `first_name` | Стандартное поле WordPress (если уже есть, ничего не делаем) |
| Фамилия | Text | `Фамилия` | `last_name` | Стандартное поле WordPress |
| Адрес доставки | Textarea | `Адрес доставки` | `delivery_address` | Полный адрес, который пользователь вводит в настройках профиля |
| Ссылка на мессенджер | Text | `Ссылка на мессенджер` | `messenger_link` | Используется в контактных данных |

> ⚠️ Обязательно использовать указанные ключи (`phone_number`, `delivery_address`, `messenger_link`), именно их читает back-end (`core/user-profile/user-profile-rest-controller.php`) и order wizard.

## 3. Отображение в админке

- Для каждого поля включаем **Show in Admin** и **Show on Add/Edit** — так менеджеры смогут видеть и править эти данные в карточке пользователя.
- При желании можно включить отображение в колонках списка пользователей (опция **Show in Admin Columns**).

## 4. Проверка

После сохранения Pod:

1. Зайти в профиль пользователя в админке и убедиться, что поля отображаются.
2. Из фронта обновить профиль (`Настройки профиля` → сохранить). Данные должны появиться в админке и возвращаться через REST (`GET /wp-json/elkaretro/v1/user/profile`).

Теперь при оформлении заказа или редактировании профиля мы можем безопасно обновлять эти поля, и они будут доступны всем подсистемам (User Profile, Order Wizard, email уведомления).
